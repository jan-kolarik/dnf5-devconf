Features

  • Lightweight as Microdnf and fully featured as DNF
    • MICRODNF install size ~ 115 MB
    • DNF install size ~ 166 MB
    • DNF5 install size ~ 114 MB
  • Integration of plugins into LIBDNF5 library
  • Fully integrated Modularity in LIBDNF5 workflows
  • New Daemon with support of Comps groups and modules
  • Improved API

Improved API

  • Safer workflows
    • Locking of configuration options
    • Checking workflows
    • Transparent errors
  • Transaction reports in structure (API)
  • Direct usage of configuration
    • Less parameters required in API
  • Better API -> less code in your project
    • All logic is in libdnf5

Optional download of filelists

Positives

  • Smaller downloads
  • Faster processing of metadata
  • Faster transaction resolvement

Negatives

  • File and provide searches
    • dnf provides /usr/lib/<something>
  • Unsatisfied dependencies
    • Using file provides is against Fedora guidelines
    • In Fedora 38
      • 72 source packages uses file provides in requires
      • 36 different file provides
      • Terminus-fonts requires: /usr/lib/kbd

RPM queries - Repoquery

DNF

  • dnf repoquery dnf
    • 0.87 s
  • dnf repoquery $(rpm -qa)
    • 4.06 s

  • dnf repoquery --whatdepends 'dnf*'
    • 82.9 s

DNF5

  • dnf5 repoquery dnf
    • 0.76 s
  • dnf5 repoquery $(rpm -qa)
    • 1.42 s

  • dnf5 repoquery --whatdepends 'dnf*'
    • 25.1 s

DNF

  • dnf upgrade $(rpm -qa) --assumeno

Adoption of DNF5 by other teams

  • DNF5 API stability
    • Stabilizing of API during Fedora 38
  • Why to be an early adopter of DNF5?
    • Application using DNF CLI
      • Test your application and report missing commands or options
    • Application using DNF, hawkey, or libdnf API
      • Help us with shaping API
      • Share with us your use cases
      • Request missing features
    • Planned removal of dnf, python3-dnf, python3-hawkey, libdnf, python3-libdnf, dnf-plugins from Fedora distribution

Changes in DNF5

  • Workflow optimization
    • Configuration locking
    • Modularity integration
    • Unified API for downloading
  • Repositories loading
    • Load and download in parallel
    • Configurable metadata
  • Configuration options
    • Stored at a single place
    • Naming preserved
      • Exceptions like --no-<option>
      • Compatibility aliases
      • strict ⟶ skip_broken, skip_unavailable
  • CLI changes
    • No optional arguments
    • Bounding options
    • Feedback welcome
  • Bash autocompletion
  • Configurable aliases

API DEMO

                        
                            import libdnf5

                            base = libdnf5.base.Base()
                            base.load_config_from_file()
                            base.setup()

                            sack = base.get_repo_sack()
                            sack.create_repos_from_system_configuration()
                            sack.update_and_load_enabled_repos(True)

                            goal = libdnf5.base.Goal(base)
                            goal.add_install('zlib')

                            transaction = goal.resolve()
                            transaction.download()
                            transaction.run()
